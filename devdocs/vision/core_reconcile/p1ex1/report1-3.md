# Phase 1.5 Report ‚ÄĒ Steps 1‚Äď3 (renderer port, fetch adapter, `nctl render hosts-intent`, parity gate)

Date: 2026-07-16. Implements [p1ex1/plan.md](plan.md) Steps 1‚Äď3. This is the first commit unit
of the plan's suggested order: everything below touches only nctl; nintent and ansible_agdev
are untouched (their deletions are Steps 4‚Äď5, after this parity result).

## What was built

### Step 1 ‚ÄĒ `nctl/src/nctl_core/hosts_intent.py`

- Ported `export_hosts_intent`, `hosts_intent_payload`, `render_hosts_intent_yml`,
  `render_hosts_intent_json`, and the private helpers from
  `nintent/nautobot_intent_catalog/ansible_inventory.py`.
- Input shape changed from `getattr`-on-ORM to the Phase 2 desired-snapshot models
  (`sources.desired.DesiredNode` / `DesiredEndpoint`): the caller passes the flat node and
  endpoint lists, and the module groups endpoints by `node_id`
  (replacing `prefetch_related("desired_endpoints")` and the `RelatedEndpoints`/list duck
  typing ‚ÄĒ `_endpoints()` and `_list()` from the source have no equivalent and were dropped).
  `_pk()` was also dropped: the snapshot models carry `id` as a plain string.
- Everything observable is unchanged: eligibility gates (`ELIGIBLE_NODE_LIFECYCLES`,
  `ELIGIBLE_NODE_TYPES` incl. `service_host`), mDNS endpoint selection (primary ‚Üí management ‚Üí
  fallback, tie-break `(endpoint_type, name)`), node sort `(slug, name)`, hostname validation
  (non-empty, no whitespace/colon), the `skip_reasons` vocabulary, the single `ssh_hosts`
  group with `nintent_*` hostvars, summary keys, and the YAML/JSON serialization options.
- Header/schema changes per plan: `# Generated by nctl` replaces the Job banner;
  `job_result_id` is dropped entirely (no `operation_id` replacement ‚ÄĒ render is synchronous,
  matching `render dnsmasq`/`render production`); `ANSIBLE_HOSTS_INTENT_SCHEMA_VERSION` ‚Üí
  `HOSTS_INTENT_SCHEMA_VERSION`, bumped `2.0` ‚Üí `3.0`. The only thing that asserted on the
  version string is the export playbook, which dies in Step 5.
- Ported `tests/test_ansible_inventory.py` (13 unittest cases) to
  `nctl/tests/test_hosts_intent.py` as pytest functions with snapshot-model fixtures. One
  source case (`test_related_manager_style_endpoints_are_supported`) is ORM-specific and has
  no equivalent; two cases were added instead: endpoint-list order independence (the plan's
  Step 2 verification ‚ÄĒ output is provably order-insensitive) and
  `include_skipped=False` behavior (counts kept, details omitted), which the old suite never
  covered.

### Step 2 ‚ÄĒ fetch adapter

- No new GraphQL query, as planned: `hosts_intent_render.build_hosts_intent_render` calls
  `sources.desired.fetch_desired_snapshot` (the Phase 2 pinned query) and feeds
  `snapshot.nodes` / `snapshot.endpoints` to the renderer. Desired-only is one GraphQL round
  trip; it deliberately does **not** go through `build_source_snapshot`, which would add the
  actual-side fetch this renderer never reads.

### Step 3 ‚ÄĒ `nctl render hosts-intent` + shared write helper

- New module `nctl/src/nctl_core/inventory_write.py`: `write_validated_inventory(yaml, path)`
  ‚ÄĒ the staged-file + `ansible-inventory --list` validation + atomic-replace mechanism,
  factored out of `production_render.write_production_artifacts`, which now calls it (error
  codes and messages unchanged: `ansible_executable_missing`, `artifact_write_failed`,
  `ansible_inventory_failed`, `ansible_inventory_invalid`). One behavioral nuance from the
  refactor: the production report JSON is now written after the inventory replace instead of
  between validation and replace; a report-write failure still returns an error.
- New module `nctl/src/nctl_core/hosts_intent_render.py` + CLI command
  `nctl render hosts-intent [--out DIR] [--json]` on the existing `render` sub-app:
  - default: inventory YAML to stdout (pipeable, matches the other render commands);
  - `--out DIR`: writes `DIR/hosts_intent.yml` via `write_validated_inventory` (previous file
    preserved on validation failure, keeping the old playbook's rescue semantics) plus
    `DIR/hosts-intent-export.json`, and prints a summary;
  - `--json`: envelope `nctl.render.hosts_intent.v1` ‚ÄĒ `data` = schema_version, summary,
    inventory, hosts, skipped, inventory_yaml, export_json.
- The plan's naming deviation stands: **`render hosts-intent`**, not the roadmap's
  `render inventory`, because `render production` (Phase 2) also renders an inventory.
- New tests: `test_cli_render_hosts_intent.py` (5 cases, mirrors the production CLI tests) and
  `test_hosts_intent_render.py` (build with faked fetch; artifact-write success/invalid/
  propagation, including previous-file preservation). `test_production_render.py`'s
  monkeypatch targets were repointed from `nctl_core.production_render.{shutil,subprocess}` to
  `nctl_core.inventory_write.{shutil,subprocess}` ‚ÄĒ same assertions, unchanged semantics.
