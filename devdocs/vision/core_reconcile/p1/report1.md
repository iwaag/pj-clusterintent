# Phase 1 Report — Step 1 (Port the dnsmasq renderer into `nctl_core`)

Date: 2026-07-14. Implements [p1/plan.md](plan.md) Step 1.

## What was built

- New module `nctl/src/nctl_core/dnsmasq.py`: ported `export_dnsmasq_records`,
  `resolve_dhcp_reservation`, `_dns_record_entry`, `_dhcp_range_entry`, the skip-reason helpers,
  `render_dnsmasq_records_conf`, `dnsmasq_export_payload`, and `render_dnsmasq_export_json` from
  `nintent/nautobot_intent_catalog/dnsmasq.py`.
  - Input access changed from `getattr`-on-ORM to plain-mapping `.get()` reads, since the Step 2
    GraphQL fetch layer will hand this module dicts, not Django model instances. `_pk` now reads
    `obj.get("id")` (GraphQL's field name) falling back to `obj.get("pk")`; `_evaluation_data`
    dropped the three-way `as_defaults()`/dict/attribute branch from the source (that existed to
    support a Django `IntentEvaluation` instance) since evaluations are also mappings on this
    side.
  - Record lines, sort keys, the `skip_reasons` vocabulary, and summary-count shapes are
    byte-identical to the source — this was verified line-by-line while porting and is also what
    the ported test suite checks (see below).
  - Header changes per plan: `# Generated by nctl` replaces `# Generated by Nautobot Intent
    Catalog`; `# operation_id: <id>` replaces `# job_result_id: <id>` (parameter renamed
    `job_result_id` → `operation_id` throughout the three render functions); schema version
    bumped from `3.0` to `4.0` (`DNSMASQ_EXPORT_SCHEMA_VERSION`). Nothing asserts on this version
    string yet — the old Job's own schema-3.0 assert lives in the `ansible_agdev` playbook and is
    deleted in Step 5, not read by this module.
- Ported `nintent`'s `tests/test_dnsmasq.py` to `nctl/tests/test_dnsmasq.py` as plain pytest
  functions (nctl's suite uses pytest, not `unittest`). Fixtures changed from
  `SimpleNamespace(...)` to plain `dict`s (`node()`, `endpoint()`, `ip_range()`,
  `mac_candidate()` factory functions), matching the mapping-based renderer. All 12 original
  cases carried over unchanged in intent: eligibility filtering, mDNS-as-metadata, record-type
  formatting and sort order, cname alias requirement, DHCP skip-reason cases (missing/ambiguous/
  invalid MAC, non-DHCP IP policy), the conf/JSON render round-trip (updated for the new header
  and `operation_id` field name), dynamic DHCP-range rendering and its skip reasons, and the JSON
  payload key ordering.

## Verification

- `uv run pytest tests/test_dnsmasq.py -q` — 12 passed.
- `uv run pytest -q` (full nctl suite) — 61 passed, 0 failures. The new module doesn't touch any
  existing code path (CLI, config, events, nautobot client, status, dumps), so this also confirms
  no regression.

exactly as described, including the header/schema-version changes the plan called out explicitly.
