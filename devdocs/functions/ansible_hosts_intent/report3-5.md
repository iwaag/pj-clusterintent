## Implemented Behavior

### Generated Inventory Root

Added a new generated/bootstrap inventory root:

```text
inventories/generated/
└── group_vars
    └── all
        ├── main.yml
        └── vault.example.yml
```

`main.yml` contains the minimal connection and Nautobot API variables needed by
the generated bootstrap inventory:

- `connection_path`
- `local_connection_host`
- `ansible_host`
- `default_user`
- `ansible_become_password`
- `nautobot_url`
- `nautobot_token`
- `nautobot_validate_certs`

`hosts_intent.yml` itself is not committed. It is expected to be generated by
the export playbook.

### Export Playbook

Added:

```text
playbooks/export_nintent_hosts_intent.yml
```

The playbook:

1. Uses Nautobot API credentials from existing vars/vault flow.
2. Finds the Nautobot Job named `Export Ansible Hosts Intent`.
3. Runs it with `include_skipped`.
4. Polls the JobResult.
5. Finds the `hosts_intent.yml` file proxy.
6. Downloads the file.
7. Validates the `# schema_version: 1.0` header.
8. Writes `inventories/generated/hosts_intent.yml`.
9. Runs `ansible-inventory --list` against the downloaded inventory.

This was not executed against Nautobot because the runtime environment is not
available here.
